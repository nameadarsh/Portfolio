Why I Built It:

I wanted to build a Retrieval-Augmented Generation system that solves a real problem instead of creating another generic chatbot.

The objective was to retrieve relevant Bhagavad Gita verses using semantic search and transform them into grounded modern-day guidance while preventing hallucinations.

Core Features:

- Semantic search over scripture
- Retrieval-Augmented Generation
- Streaming responses
- Real-time Text-to-Speech
- Support for 11 Indian languages
- Modern responsive frontend
- FastAPI backend
- Background warmup
- Optimized latency

Architecture:

Frontend: React, TypeScript
Backend: FastAPI
Retrieval: FAISS vector search
Generation: LLM reasoning over retrieved passages
Streaming: Server Sent Events

Major Engineering Decisions:
- Used semantic retrieval instead of keyword search.
- Separated retrieval from generation.
- Implemented streaming responses to reduce perceived latency.
- Used background warm-up to reduce cold start delays.
- Designed APIs to remain modular for future model replacement.

Result:
Approximately 40% lower average response latency.
Most Important Learning
Building an AI product involves much more than choosing a model.
System architecture, retrieval quality, latency and user experience matter just as much.
